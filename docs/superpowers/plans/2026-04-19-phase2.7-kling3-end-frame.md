# Phase 2.7 — Kling 3.0 Pro Upgrade + Start/End Frame Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the Kling provider from `kling-v2-master` to `kling-v3-pro` and activate start+end frame video generation so the AI knows where the camera starts AND ends. Director picks a paired end photo when two uploaded photos reasonably connect (e.g., drone-wide → facade-close, kitchen-entrance → island-close, orbital angle-A → angle-B). When no pair exists, the pipeline falls back to a center-crop variation of the same photo as the end frame so push_ins still get the quality lift.

**Architecture:** Scenes and Lab iterations gain `end_photo_id` + `end_image_url` columns. Photo analyzer stays single-photo. Director, which sees all photos + analyses, picks the optional end photo per scene when it detects a good pair. Kling provider receives `startImageUrl` + `endImageUrl` in `GenerateClipParams` and sends them to Kling 3.0 Pro's `image_list` payload with `type: "start_frame"` and `type: "end_frame"`. A `lib/services/end-frame.ts` helper resolves the end URL: returns the paired photo's URL when set, else generates a center-crop variant via `sharp`, uploads to Supabase Storage, returns the new URL. Runway and Luma continue receiving single-frame calls unchanged — only Kling-routed scenes get end-frame behavior.

**Tech Stack:** TypeScript (ESM), Vercel Serverless Functions (Node 20), Supabase Postgres + Storage, `sharp` (already in deps), Kling Edit API v3 (`/v1/videos/image2video` endpoint updated payload), existing JWT auth.

**Scope boundaries (do NOT build in this plan):**

- No Runway or Luma changes (neither supports end-frame; keep them as-is for orbital exteriors + parallax)
- No Wan 2.7 integration (deferred — only revisit if cost becomes dominant)
- No Gemini / Claude image-editing end-frame generation (deferred — crop-variation fallback is good enough)
- No UI for manual pair selection in Lab (director chooses; manual override is a later plan)
- No re-rating / re-learning on old v2-master iterations (existing ratings still valid as signal; new model produces new outputs)

Create `lib/services/end-frame.ts`:

```ts
import sharp from "sharp";
import { getSupabase } from "../client.js";

export interface EndFrameInputs {
  startPhotoUrl: string;
  endPhotoUrl?: string | null;
}

// Resolves the URL to hand to Kling as the `end_frame` keyframe.
//   1. If the director paired a real end photo, use that.
//   2. Otherwise, synthesize a "push-in" end frame by cropping the
//      center 75% of the start photo and uploading it to Supabase
//      Storage. Kling interpolates between the original and the crop,
//      producing a gentle forward push.
// A test seam (__setCropFnForTests) lets unit tests bypass the real
// sharp + Supabase upload without mocking them directly.
export async function resolveEndFrameUrl(input: EndFrameInputs): Promise<string> {
  if (!input.startPhotoUrl) {
    throw new Error("resolveEndFrameUrl: startPhotoUrl is required");
  }
  if (input.endPhotoUrl) {
    return input.endPhotoUrl;
  }
  const cropFn = cropOverride ?? generateCenterCropEndFrame;
  return cropFn(input.startPhotoUrl);
}

// Default implementation: fetch the start photo, center-crop to 75%,
// upload back to Supabase Storage under a deterministic path. The
// resulting URL is cached-by-hash in the bucket so repeat renders of
// the same photo reuse the same crop.
async function generateCenterCropEndFrame(startPhotoUrl: string): Promise<string> {
  const res = await fetch(startPhotoUrl);
  if (!res.ok) throw new Error(`Fetch start photo failed: ${res.status}`);
  const buffer = Buffer.from(await res.arrayBuffer());

  const image = sharp(buffer);
  const meta = await image.metadata();
  const w = meta.width ?? 1024;
  const h = meta.height ?? 576;
  // 75% centered crop produces a noticeable but stable "destination".
  const cropW = Math.round(w * 0.75);
  const cropH = Math.round(h * 0.75);
  const left = Math.round((w - cropW) / 2);
  const top = Math.round((h - cropH) / 2);
  const cropped = await image
    .extract({ left, top, width: cropW, height: cropH })
    .resize(w, h, { fit: "cover" })  // scale back up so Kling sees matching dimensions
    .jpeg({ quality: 90 })
    .toBuffer();

  const hash = simpleHash(startPhotoUrl);
  const path = `end-frames/${hash}.jpg`;
  const supabase = getSupabase();
  const { error: uploadErr } = await supabase.storage
    .from("property-photos")
    .upload(path, cropped, { contentType: "image/jpeg", upsert: true });
  if (uploadErr) throw new Error(`Upload crop end-frame failed: ${uploadErr.message}`);

  const { data: publicUrlData } = supabase.storage.from("property-photos").getPublicUrl(path);
  return publicUrlData.publicUrl;
}

// FNV-1a 32-bit. Good enough for deterministic filenames; not crypto.
function simpleHash(input: string): string {
  let h = 2166136261;
  for (let i = 0; i < input.length; i++) {
    h ^= input.charCodeAt(i);
    h = (h * 16777619) >>> 0;
  }
  return h.toString(16).padStart(8, "0");
}

// Test seam — swap the real crop/upload for a fake in unit tests.
let cropOverride: ((url: string) => Promise<string>) | null = null;
export function __setCropFnForTests(fn: ((url: string) => Promise<string>) | null): void {
  cropOverride = fn;
}
```

```text
When to set end_photo_id:
```

```ts
import { resolveEndFrameUrl } from "./services/end-frame.js";

const director = iter.director_output_json as { end_photo_id?: string } | null;
const startPhotoUrl = session.image_url;          // Lab session's uploaded image
let endPhotoUrl: string | null = null;
if (director?.end_photo_id) {
  // Lab sessions are single-image — if the director paired an end photo
  // it must be another Lab session the user wants to pair with. Look up
  // by id; skip silently if not found so Lab keeps working.
  const { data: endSession } = await supabase
    .from("prompt_lab_sessions")
    .select("image_url")
    .eq("id", director.end_photo_id)
    .maybeSingle();
  endPhotoUrl = endSession?.image_url ?? null;
}

const endImageUrl = await resolveEndFrameUrl({
  startPhotoUrl,
  endPhotoUrl,
});

// Persist onto the iteration:
await supabase
  .from("prompt_lab_iterations")
  .update({
    end_photo_id: director?.end_photo_id ?? null,
    end_image_url: endImageUrl,
  })
  .eq("id", iter.id);

// Pass through to the router:
const params: GenerateClipParams = {
  // ... existing ...
  sourceImageUrl: startPhotoUrl,
  endImageUrl,
};
```
